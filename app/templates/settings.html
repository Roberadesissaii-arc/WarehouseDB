    JSON format

    Each product in the products array needs:

    • name — item name
    • sku — stock keeping unit
    • quantity — units on shelf
    • age_days — optional, how long it has been in stock
    • warehouse — site name
    • aisle — section / aisle name
    • bay — shelf / bay code

    Example file on disk (not loaded automatically): json/products.sample.json

    Robot integration

    A real robot connects over HTTP using these endpoints — no extra services. Same server as the board.

    HTTP Direct on LAN
    1 Server host
    X-Robot-Code Device auth header
    Staff logged-in browser session Robot X-Robot-Code header Open no login required

    Pairing

    POST /api/robots/pair Staff: link a robot's 6-digit screen code to a name Staff
    POST /api/robots/claim Robot: confirm pairing with the same code Open
    GET /api/robots/<id>/pair-status Poll while waiting for the robot to connect Staff

    Robot device

    GET /api/robots/<id>/tasks Robot polls work — header X-Robot-Code Robot
    PUT /api/tasks/<id> Report progress (queued → in progress → done) Robot
    PUT /api/robots/<id> Edit display name / home bay (not live status) Staff

    Lookups

    GET /api/items/<id> Resolve an item to its exact location path Staff
    GET /api/tree Full warehouse → section → bay hierarchy Staff

    Base URL is this server. The full endpoint list is in the project README.md and the staff guide.
